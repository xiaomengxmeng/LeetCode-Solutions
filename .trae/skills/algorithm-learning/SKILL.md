## 📝 深度学习方法论（5步法）

### 🔍 Step 1: 问题分析（5分钟）
- **输入输出**：明确函数签名和返回值
- **约束条件**：时间/空间复杂度要求，数据范围
- **示例分析**：手动走2-3个例子，寻找规律
- **提问**：向AI提出关键问题：
  - 这道题可以用哪些方法解决？
  - 最优解的时间复杂度是多少？
  - 有哪些边界情况需要注意？

### 💡 Step 2: 思路设计（10分钟）
- **暴力解法**：先想最简单的方法（通常O(n²)或更高）
- **优化方向**：识别重复计算 → 用空间换时间
- **选择数据结构**：根据操作特点选择合适的数据结构
- **画图辅助**：用图表展示算法流程
- **伪代码**：先写逻辑，再写语法

### ⚙️ Step 3: 代码实现（15分钟）
- **清晰变量命名**：避免 `i, j, k`，使用有意义的名称
- **模块化**：将大问题拆分成小函数
- **边界处理**：空值、单元素、极端情况
- **代码规范**：遵循项目编码风格

### 🔧 Step 4: 优化反思（10分钟）
- **复杂度分析**：实际运行时间和空间占用
- **优化可能性**：能否降低时间/空间复杂度？
- **替代方案**：还有其他解法吗？各有什么优劣？
- **错误总结**：踩了哪些坑？为什么会错？

### 🚀 Step 5: 知识扩展（5分钟）
- **变体思考**：如果修改题目条件会怎样？
- **模式提取**：这道题属于哪类问题？通用解法是什么？
- **关联题目**：类似的题目有哪些？（如Two Sum → Three Sum → Four Sum）
- **费曼检验**：能否用简单语言解释给初学者听？

## 📊 每日学习流程

### 🌅 晨间启动（15分钟）
1. **回顾昨日**：快速浏览昨天记录的DailyRecord.md
2. **今日目标**：确定今天要做的1-2道题（质量>数量）
3. **预习知识点**：阅读相关数据结构/算法的理论知识

### 🎮 核心练习（45-60分钟/题）
按照**深度学习方法论的5步法**完成每道题目

### 🌙 晚间总结（15分钟）
1. **调用本Skill的记录功能**：将今日学习内容写入DailyRecord.md
2. **自我评估**：今天真正理解了吗？还是只是记住了答案？
3. **明日规划**：根据今天的学习情况调整明天的计划

### 示例记录

```markdown
## 2026-04-13 | [Easy] Two Sum (1)

### 🎯 核心思路
使用哈希表存储已遍历元素及其索引，对于每个元素 `nums[i]`，检查 `target - nums[i]` 是否在哈希表中。时间复杂度 O(n)，空间复杂度 O(n)。

### 🔑 关键代码片段
```cpp
unordered_map<int, int> numToIndex;
for (int i = 0; i < nums.size(); i++) {
    int complement = target - nums[i];
    if (numToIndex.count(complement)) {
        return {numToIndex[complement], i};
    }
    numToIndex[nums[i]] = i;
}
return {};
```

### 📚 学到的知识点
- 哈希表的 O(1) 查找特性可以用于"两数之和"类问题
- "以空间换时间"是常用的优化策略
- 遍历时同时构建查找表是一种常见模式

### ⚠️ 易错点 & 反思
- ❌ 一开始用了双重循环 O(n²)，忘记考虑哈希表优化
- 💡 以后看到"查找两个数满足某条件"，优先考虑哈希表

### 🔗 关联题目
- Three Sum (15) - 扩展到三个数
- Four Sum (18) - 进一步扩展
```

1. **重做题目**：不看答案，独立完成
2. **优化代码**：尝试用更优的方法解决
3. **讲解思路**：用口头或文字解释解题过程
4. **对比进步**：查看之前的记录，看是否有新的理解

### 🎯 AI的行为准则：

✅ **应该做的**：
- 循序渐进地引导，不要一次性灌输太多信息
- 多问"你觉得呢？"来激发主动思考
- 提供可视化的解释（ASCII图、步骤分解）
- 鼓励用户先用暴力法，再优化

❌ **不应该做的**：

- 不要跳过分析过程直接讲最优解
- 不要忽视用户的疑问，即使看起来很简单
- 不要一次推荐超过2道题目（贪多嚼不烂）

## 💡 学习建议

### 🎯 关于"记不住"的问题
很多同学觉得"学了就忘"，这是因为：
1. **只看不练**：看懂≠会写，必须亲手实现
2. **浅尝辄止**：只记住答案套路，不理解为什么
3. **缺乏重复**：没有按照遗忘曲线及时复习
4. **孤立学习**：没有建立知识之间的联系

### ✨ 解决方案
- **费曼学习法**：学完一道题，尝试讲给别人听
- **模式抽象**：不只记这道题怎么做，要提炼出通用模式
- **间隔重复**：利用复习机制定期巩固
- **项目实战**：在实际项目中运用算法知识
