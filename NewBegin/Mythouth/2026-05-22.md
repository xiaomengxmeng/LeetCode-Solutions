## [Easy] 二叉树的直径 (543)
### 🤔 我的原始思路
[在此记录你的初步想法]
root 为起点的直径 = max (左子树高度 + 右子树高度)
"直径"和"最大深度"是不同的概念
 如果直径不穿过根节点,那么最大直径在最高的左,右子树中
 穿过根节点,那么最大直径在左子树高度 + 右子树高度 + 1

 每个节点作为最高点 时的路径 = 左子树高度 + 右子树高度 + 1


### 🔍 我的思考过程
- 直径 = 边数，不是节点数
- 每个节点作为最高点时的直径（边数）= 左子树高度 + 右子树高度
- 用全局变量 ans 在后序遍历中收集最大值
### 💡 对比收获
- **与 104 Max Depth 的关联**：543 的 dfs 返回值就是 104 的最大深度，只是在计算过程中顺便更新了直径
- **与 110 Balanced Tree 的模式一致**：都是在后序遍历中"顺便"记录全局信息
- **"高度伪装"技巧**：函数返回高度，用全局变量收集直径——一套模板解决多类问题

## [Medium] Longest Univalue Path (687)
### 🤔 我的原始思路
[在此记录你的初步想法]
543 的模板是 ans = max(ans, L + R) ，687 在什么时候才能用 L + R 更新答案？
- 当左子树和右子树的值都相同时，才能用 L + R 更新答案
dfs 返回长度和节点值
路径拼接 ：什么情况下左子树路径 + 当前节点 + 右子树路径能连成一条通路的同值路径
3者的节点值都相同
### 🔍 我的思考过程
- 与 543 模板完全一致，只是多了"值判断"：孩子值不同则贡献清零
- 左右独立判断、独立清零
- ans 全局收集，返回贡献给父节点
### 💡 对比收获
- **从 543 → 687 是加条件而非改结构**：模板不变，只加了两行 if 判断
- **短路求值**：`root->left && root->left->val != root->val` 兼顾安全和功能
- **"后序遍历 + 全局变量" 模式**：110 Balanced Tree → 543 Diameter → 687 Longest Univalue，一脉相承
