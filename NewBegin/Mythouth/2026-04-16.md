## [Medium] Longest Substring Without Repeating Characters (3)
### 🤔 我的原始思路
[在此记录你的初步想法]
变长滑动窗口 
数组记录窗口中的字符
重复字符窗口右边界移动？ 
如何判断窗口中是否有重复字符？ 用数组记录窗口中的字符，若数组中存在该字符，说明窗口中存在重复字符，否则不存在重复字符
遇到重复字符时，窗口右边界移动，直到窗口中不存在重复字符
窗口大小何时变化？ 
窗口左边界如何移动


### 🔍 我的思考过程

**Step 1: 问题分析**
1. 核心要求：找最长不重复子串
2. 与 Find Anagrams 的区别：窗口大小可变
3. 关键：遇到重复字符时，左边界如何移动

**Step 2: 设计方案（变长滑动窗口）**
- **右边界**：不断向右扩展
- **左边界**：遇到重复字符时跳跃更新
- **pos 数组**：记录每个字符最后出现的位置
- **更新规则**：
  - 若 `pos[s[i]] >= left`，说明重复字符在窗口内
  - `left = pos[s[i]] + 1`，跳到重复字符之后

**Step 3: 代码实现**
```cpp
int lengthOfLongestSubstring(string s) {
    vector<int> pos(128, -1);  // 记录字符最后位置
    int left = 0;              // 窗口左边界
    int maxLen = 0;            // 最大长度
    
    for (int i = 0; i < s.size(); i++) {
        if (pos[s[i]] >= left) {
            left = pos[s[i]] + 1;  // 跳到重复字符之后
        }
        pos[s[i]] = i;
        maxLen = max(maxLen, i - left + 1);
    }
    return maxLen;
}
```

**Step 4: Bug 修复**
- ❌ 原始代码：`left = i` — 跳到了当前位置，窗口变空
- ✅ 修复后：`left = pos[s[i]] + 1` — 跳到重复字符之后

**示例追踪**：
```
s = "abcabcbb"
i=3: 'a' → pos=0 >= left(0), left=1, 窗口="bca"
i=4: 'b' → pos=1 >= left(1), left=2, 窗口="cab"
```

### 💡 对比收获

**与 Find Anagrams 的对比**：
| 维度 | Find Anagrams (438) | Longest Substring (3) |
|------|---------------------|----------------------|
| 窗口大小 | 固定 | 可变 |
| 左边界移动 | 每次移动1步 | 遇重复时跳跃 |
| 数据结构 | count 数组 | pos 数组 |

**新学到的知识点**：
1. **变长滑动窗口**：左右边界独立移动
2. **跳跃更新**：`left = pos[c] + 1`，O(1) 跳过重复字符
3. **pos 数组技巧**：记录最后位置，快速判断是否在窗口内
4. **窗口大小计算**：`i - left + 1`

## [Medium] Permutation in String (567)
### 🤔 我的原始思路
这道题只需要判断异位词是否存在，几乎一样
核心 固定大小滑动窗口 判断窗口是否是s1的排列
关键 一样处理进入窗口和离开窗口的字符后是否匹配 

### 🔍 我的思考过程

**Step 1: 问题分析**
1. 核心要求：判断 s2 是否包含 s1 的排列
2. 与 Find Anagrams 的关系：几乎完全相同
3. 区别：只需要返回 true/false，不需要所有索引

**Step 2: 设计方案**
- **窗口大小**：固定为 s1.size()
- **核心逻辑**：与 Find Anagrams 完全相同
- **优化**：找到匹配立即返回 true

**Step 3: 代码实现**
```cpp
bool checkInclusion(string s1, string s2) {
    if (s1.size() > s2.size()) return false;
    
    vector<int> count(26);
    for (char c : s1) count[c - 'a']++;
    
    int s1_count = 0;
    for (int c : count) if (c != 0) s1_count++;
    
    int match = 0;
    
    // 初始化窗口
    for (int i = 0; i < s1.size(); i++) {
        count[s2[i] - 'a']--;
        if (count[s2[i] - 'a'] == 0) match++;
        else if (count[s2[i] - 'a'] == -1) match--;
    }
    if (match == s1_count) return true;
    
    // 滑动窗口
    for (int i = s1.size(); i < s2.size(); i++) {
        // 进入字符
        count[s2[i] - 'a']--;
        if (count[s2[i] - 'a'] == 0) match++;
        else if (count[s2[i] - 'a'] == -1) match--;
        
        // 离开字符
        count[s2[i - s1.size()] - 'a']++;
        if (count[s2[i - s1.size()] - 'a'] == 0) match++;
        else if (count[s2[i - s1.size()] - 'a'] == 1) match--;
        
        if (match == s1_count) return true;
    }
    return false;
}
```

**Step 4: Bug 修复**
- ❌ 离开字符时检查 `== -1`，应该是 `== 1`
- ✅ `count[c]++` 后，从 0→1 表示从匹配变成不足

### 💡 对比收获

**与 Find Anagrams 的对比**：
| 维度 | Find Anagrams (438) | Permutation in String (567) |
|------|---------------------|----------------------------|
| 返回值 | 所有起始索引 | 是否存在 |
| 核心逻辑 | 完全相同 | 完全相同 |
| 优化 | 无 | 找到即返回 |

**新学到的知识点**：
1. **代码复用**：相同模式可以解决不同问题
2. **提前返回优化**：找到答案立即返回，减少不必要的遍历
3. **离开字符的 match 更新**：`count[c]++` 后检查 `== 1`
