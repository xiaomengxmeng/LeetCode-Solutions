## [Medium] 01 Matrix (542)

### 🤔 我的原始思路

[在此记录你的初步想法]

多源 BFS 源 是1

初始化 所有边界1的位置

边界 1 距离值初始化为 1

### 🔍 我的思考过程

源 是 0
0的初始距离 是 0
1的初始距离 设为未访问标记（-1 或 INF）
BFS 从 0 向外扩散，遇到未被访问的 1 时赋值

### 💡 对比收获

- **源的选择是关键**：一开始想源是 1，但发现每个 1 独立 BFS 太慢；改成源是 0，所有 0 同时入队，一次 BFS 搞定
- **与 994 Rotting Oranges 对比**：994 是烂橘子感染新鲜橘子（1→2），542 是 0 扩散找 1 → 两题核心都是多源 BFS
- **INT_MAX 作为未访问标记**：比用 -1 更巧妙，因为 INT_MAX 天然大于任何距离，赋值时 `res[nx][ny]=res[x][y]+1` 无溢出风险
- **逗号表达式的坑**：`vector<vector<int>> res=(m, vector<int>(n))` 里的逗号是运算符，不是构造函数参数

## [Medium] As Far from Land as Possible (1162)
### 🤔 我的原始思路
和 01 Matrix 542 一样
最后返回 res 中的最大值
源 是 1
全是 0 或全是 1 返回 -1
### 🔍 我的思考过程
源是 1（陆地），所有陆地同时入队（距离=0）
海洋初始化为 INF，BFS 向外扩散
最后找所有海洋中的最大值
### 💡 对比收获
- **与 542 同源不同向**：542 源是 0 向外扩散到 1，1162 源是 1 向外扩散到 0，模板完全对称
- **BFS 分层计时的两种写法**：542 用 `res[nx][ny]=res[x][y]+1` 记录每个格子的距离；1162 用 `step` 按层累计更简洁
- **原地修改 vs 额外数组**：1162 可以直接改原数组 `grid[nx][ny]=1` 标记已访问，比 542 额外开 res 省空间
- **一维化压缩**：`i*m+j` 入队比 `pair<int,int>` 更省内存，适合 queue 操作
