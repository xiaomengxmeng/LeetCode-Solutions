## [Medium] Find Minimum in Rotated Sorted Array (153)
### 🤔 我的原始思路
[在此记录你的初步想法]
没有旋转时，第一个就是最小值
旋转后 第一个小于前面的值就是最小值 也小于它后面的值
nums[mid] > nums[0] mid 在左段 min 在mid右边
判断 nums[mid] > nums[0]  往右边收缩 反之 左

### 🔍 我的思考过程
- 比 33 简单：只用判断 mid 在左段还是右段，一个条件解决
- `nums[mid] > nums[right]` → mid 在左段，最小值在右边 → left = mid + 1
- `nums[mid] <= nums[right]` → mid 在右段，最小值在 mid 或左边 → right = mid
- 用 while(left < right) 替代 while(left <= right)
- 需要小心 right 的初始值避免越界

### 💡 对比收获
- **33 vs 153**：33 需要双分支判断，153 只需一个条件；33 找目标值下标，153 找最小值
- **和 nums[right] 比较的妙处**：无旋转时也正确，不像 nums[0] 会出问题
- **while(left < right) 模式**：适合"找极值"类二分，退出时 left==right 就是答案

## [Medium] Search a 2D Matrix (74)
### 🤔 我的原始思路
[在此记录你的初步想法]
矩阵的特性（每行升序 + 每行第一个大于上一行最后一个）
拉平成一个一维数组是升序的数组
mid 对应的(row,col)  一维下标 mid
    → row = mid / n  （n 是列数）
    → col = mid % n
直接一次二分
### 🔍 我的思考过程
- 矩阵拉平成一维数组，标准二分解决
- mid → (row, col) = (mid / n, mid % n)
- 还写了个 Z 字形搜索（从右上角出发）作为备选

### 💡 对比收获
- **一次二分 vs 两次二分**：拉平后一次二分就能搞定，比先找行再找列更简洁
- **Z 字形搜索（240 题的解法）**：从右上角出发，大了左移，小了下移——这个思路对 74 也适用

## [Medium] Koko Eating Bananas (875)
### 🤔 我的原始思路
[在此记录你的初步想法]
canEat(K)  是否能吃掉所有香蕉
求和count= piles[i]/K +是否有余数 有 +1 否 0

### 🔍 我的思考过程
- "二分答案"模式：二分速度 K，用 canEat(K) 验证
- K 的范围：1 ~ max(piles)
- 验证公式：(p + K - 1) / K 实现向上取整
- while(left < right)，可行则收缩 right，否则收缩 left

### 💡 对比收获
- **新二分模式**：之前都是"在有序数组里搜目标值"，875 是"二分答案 + 验证"
- **向上取整公式**：(p + K - 1) / K 比 if(p%K) 更简洁
- **max_element**：`*max_element(piles.begin(), piles.end())` 一行拿最大值
