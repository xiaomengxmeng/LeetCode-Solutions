## [Medium] 最长公共子序列 (1143)

### 🤔 我的原始思路
- dp[i][j] 表示两个字符串以对应位置结尾的最长 LCS 的长度
- text1[i] == text2[j] 时这个字符一定在 LCS 中
- dp 表需要处理边界，建 dp(m+1, vector<int>(n+1, 0))，下标从 1 开始

### 🔍 我的思考过程
1. **状态定义**：dp[i][j] = text1[0..i-1] 和 text2[0..j-1] 的 LCS 长度（用 size+1 处理边界）
2. **相等时**：text1[i-1] == text2[j-1] → dp[i][j] = dp[i-1][j-1] + 1
3. **不相等时**：dp[i][j] = max(dp[i-1][j], dp[i][j-1]) — 跳过 text1[i-1] 或 text2[j-1]
4. **边界**：dp[0][*] = dp[*][0] = 0，空字符串没有公共子序列
5. **初始化**：用 `vector<vector<int>> dp(m+1, vector<int>(n+1, 0))`

### 💡 对比收获
- **LIS → LCS**：从一维"以 i 结尾"扩展到二维"前 i 和前 j"，两个不同的"结尾"视角
- **新的转移模式**：相等时左上+1，不等时取左/上最大值 — 不同于 LIS 的遍历所有 j
- **size+1 技巧**：统一处理边界，避免 i-1/j-1 越界判断
- **与 62/63 二维路径对比**：62 只能向右/下走，LCS 可以"跳过"字符，转移不同
- **空间优化**：二维 DP 只依赖三格（上、左、左上），可优化为一维滚动数组，空间 O(n)

## [Medium] 最长回文子串 (5)

### 🤔 我的原始思路
子串必须连续 
以每个字符开始扩散找到所有回文子串
dp[i][j]表示s[i,j]为回文子串

### 🔍 我的思考过程
1. **先写中心扩散**：expand(l, r) 向两边扩散，while 结束后回文范围是 [left+1, right-1]
2. **踩坑**：expand 返回 {left, right} 导致 substr 取错，修正为 {left+1, right-1}
3. **对比写法**：更简洁的写法是 expand 直接返回长度 (r - l - 1)，用 start + maxLen 记录
4. **区间 DP 留到明天**：按长度从小到大遍历，dp[i][j] 为 bool

### 💡 对比收获
- **中心扩散法**：O(n²) 时间，O(1) 空间，面试常用
- **substring vs subsequence**：516 可跳字符→LCS 转化，5 必须连续→中心扩散
- **区间 DP 遍历顺序**：按长度从小到大（和 LCS 按下标顺序不同！）
- **三种回文 DP 排布**：
  - 516 LPS：dp[i][j] = 长度，可跳字符
  - 5 LPS：dp[i][j] = bool，连续
  - 647 回文计数：bool dp + count
