## \[Medium] Construct Binary Tree from Preorder and Inorder Traversal (105)

### 🤔 我的原始思路

前序第一元素是根节点，中序中根节点前的是左子树，后的是右子树
左子树长度 = 中序中根的位置 - 中序起始
用这个长度在前序中划分左右子树区间
递归重复，map 缓存中序值索引

### 🔍 我的思考过程

1. 前序第一个元素 = 根节点，中序找到根的位置 = mid
2. mid 把中序切为左右两半，分别对应左右子树
3. 前序指针 pre\_idx 传引用，每取一个根就后移一位
4. 递归时先构建左子树再构建右子树（因为前序是根→左→右）
5. 提前用 unordered\_map 缓存 值→索引，O(1) 定位根

### 💡 对比收获

代码正确 ✅

1. **前序定根，中序分左右** — 构造题最核心的口诀
2. 用 `in_left > in_right` 作为递归终止条件，判断当前中序区间是否有效
3. `pre_idx` 传引用是关键，保证全局递增不回溯
4. 必须先构建左子树后构建右子树 — 前序顺序决定的
5. 哈希表缓存中序索引是标准优化手法

## [Medium] Path Sum II (113)

### 🤔 我的原始思路

关键在于记录路径

如何记录路径：用 vector<int> path 记录当前路径
什么时候把路径加入结果集：叶子节点且和为 targetSum
回溯时怎么取消对路径的修改：path.pop_back()

### 🔍 我的思考过程

1. DFS 深度优先搜索
2. 关键：入栈和出栈路径节点（push_back / pop_back 配对）
3. 只有叶子节点（左右都为空）且 sum == 0 时才存路径
4. path 传引用，每次递归完左右子树后 pop_back 回溯
5. 与 112 的区别：112 只要 bool 返回值，113 要所有路径

### 💡 对比收获

代码正确 ✅

1. **有加就有减**：push_back 和 pop_back 必须配对出现
2. 回溯是递归的"撤销操作"，保证 path 始终只包含当前路径上的节点
3. `res.push_back(path)` 存的是副本，后续 pop 不影响已存的路径
4. 和 112 对比学习效果好：一个 bool 回溯，一个收集全部路径
