## [Easy] Balanced Binary Tree (110)

### 🎯 核心思路
自底向上递归，用 -1 标记不平衡。在一次遍历中同时计算高度和判断平衡。如果发现不平衡，立即返回 -1 向上传递。

### 🔑 关键代码片段
```cpp
// 递归版本（推荐）
int getHeight(TreeNode* root) {
    if (!root) return 0;
    int left = getHeight(root->left);
    int right = getHeight(root->right);
    if (left == -1 || right == -1 || abs(left - right) > 1)
        return -1;
    return max(left, right) + 1;
}

bool isBalanced(TreeNode* root) {
    return getHeight(root) != -1;
}

// 迭代版本（后序遍历）
bool isBalanced(TreeNode* root) {
    if (!root) return true;
    stack<TreeNode*> stk;
    unordered_map<TreeNode*, int> height;
    stk.push(root);
    while (!stk.empty()) {
        TreeNode* curr = stk.top();
        bool leftDone = !curr->left || height.count(curr->left);
        bool rightDone = !curr->right || height.count(curr->right);
        if (leftDone && rightDone) {
            stk.pop();
            int lh = curr->left ? height[curr->left] : 0;
            int rh = curr->right ? height[curr->right] : 0;
            if (abs(lh - rh) > 1) return false;
            height[curr] = max(lh, rh) + 1;
        } else {
            if (curr->right) stk.push(curr->right);
            if (curr->left) stk.push(curr->left);
        }
    }
    return true;
}
```

### 📚 学到的知识点
- **自底向上**：先处理子树，再处理当前节点
- **-1 标记法**：用特殊值标记不平衡，避免重复计算
- **后序遍历迭代**：用栈 + 哈希表模拟递归

### ⚠️ 易错点 & 反思
- ❌ 每个节点重新计算高度，导致 O(n²)
- ✅ 用 -1 标记，一次遍历完成
- 💡 **技巧**：后序遍历适合需要先知道子树结果的问题

## [Easy] Path Sum (112)

### 🎯 核心思路
递归遍历，每次减去当前节点值。在叶子节点判断剩余值是否为 0。用 || 连接左右子树的递归结果。

### 🔑 关键代码片段
```cpp
bool hasPathSum(TreeNode* root, int targetSum) {
    if (!root) return false;
    targetSum -= root->val;
    if (targetSum == 0 && !root->left && !root->right)
        return true;
    return hasPathSum(root->left, targetSum) || hasPathSum(root->right, targetSum);
}
```

### 📚 学到的知识点
- **路径定义**：根节点到叶子节点的完整路径
- **叶子节点判断**：`!root->left && !root->right`
- **递归减值**：每次减去当前节点值，简化判断

### ⚠️ 易错点 & 反思
- ❌ 只判断 `targetSum == 0`，忘记检查是否是叶子节点
- ❌ 递归调用没有返回值
- ✅ 同时判断：剩余值为 0 + 是叶子节点
