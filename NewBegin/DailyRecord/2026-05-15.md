## [Medium] Binary Tree Right Side View (199)

### 🎯 核心思路
两种解法：
- **BFS**：层序遍历，每层从左到右遍历，取每层的最后一个节点（`i == size-1`）
- **DFS**：先右后左递归，利用 `depth == ans.size()` 判断每层第一个被访问的节点，即为右视图可见节点

### 🔑 关键代码片段
```cpp
// BFS 版
vector<int> rightSideView(TreeNode* root) {
    vector<int> ans;
    if (!root) return ans;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            TreeNode* node = q.front(); q.pop();
            if (i == size - 1) ans.push_back(node->val);
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
    }
    return ans;
}

// DFS 版（更简洁）
vector<int> rightSideView(TreeNode* root) {
    vector<int> ans;
    dfs(root, 0, ans);
    return ans;
}
void dfs(TreeNode* root, int depth, vector<int>& ans) {
    if (!root) return;
    if (depth == ans.size()) ans.push_back(root->val);
    dfs(root->right, depth + 1, ans);
    dfs(root->left,  depth + 1, ans);
}
```

### 📚 学到的知识点
- **两种视角都可行**：BFS 和 DFS 都能解决同一问题，各有优劣
- **`depth == ans.size()` 技巧**：DFS 先右后左时，每层第一个被访问的节点就是该层最右节点
- **BFS vs DFS 空间对比**：BFS 用队列 O(n)，DFS 用递归栈 O(h)，平衡树 DFS 更省内存

### ⚠️ 易错点 & 反思
- ❌ BFS 中忘记判断 `i == size - 1` → 会把每层所有节点都加入结果
- ❌ DFS 中如果不先右后左 → 看到的是左视图而非右视图
- 💡 **二者对比**：BFS 直观易懂，DFS 代码更简洁优雅

## [Medium] Lowest Common Ancestor of a Binary Tree (236)

### 🎯 核心思路
**后序遍历（左右根）**。递归查找 p 和 q：
- 如果当前节点是 p 或 q，直接返回
- 左子树和右子树各返回查找结果
- 两边都非空 → 当前节点是 LCA
- 只有一边非空 → 把结果往上传递

### 🔑 关键代码片段
```cpp
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root == p || root == q) return root;
    TreeNode* left = lowestCommonAncestor(root->left, p, q);
    TreeNode* right = lowestCommonAncestor(root->right, p, q);
    if (!left) return right;
    if (!right) return left;
    return root;
}
```

### 📚 学到的知识点
- **后序遍历解决祖先问题**：先处理左右子树，根据结果决定当前节点
- **递归函数的三种返回**：root 是 p/q → 返回 root；左右各一个 → 返回 root；单侧有 → 返回该侧
- **236 vs 235 区别**：235 用 BST 值判断（单路递归），236 必须搜两边（全遍历）

### ⚠️ 易错点 & 反思
- ❌ 忘记 `!root` base case → 空树直接返回 nullptr
- ❌ 搞混返回值含义：不是"找到 p/q 了吗"，而是"找到的 LCA 是什么"
- 💡 虽然要搜两边，但一旦找到 p 或 q 就提前返回，不需要继续往下搜
