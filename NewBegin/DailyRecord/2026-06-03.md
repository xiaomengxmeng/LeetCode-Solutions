## [Medium] Binary Tree Level Order Traversal (102)
### 🎯 核心思路
BFS 队列 + 按层处理：每层先记 `size = q.size()`，用 for 循环取完当前层。
### 🔑 关键代码片段
```cpp
vector<vector<int>> res;
queue<TreeNode*> q;
if (!root) return res;
q.push(root);
while (!q.empty()) {
    int size = q.size();
    vector<int> level;
    for (int i = 0; i < size; i++) {
        TreeNode* front = q.front(); q.pop();
        level.push_back(front->val);
        if (front->left) q.push(front->left);
        if (front->right) q.push(front->right);
    }
    res.push_back(level);
}
return res;
```
### 📚 对比原解法
**一致** ✅ — 但刚开始忘了 `if(!root)` 和 `q.pop()`，补上后一次过
### ⚠️ 易错点 & 反思
- 空树不检查 → push nullptr 💥
- `q.pop()` 忘写 → 死循环
- `size = q.size()` 必须在 for 之前取，不能在循环条件里判断

## [Medium] Validate Binary Search Tree (98)
### 🎯 核心思路
尝试了 **两种方法**：
- **范围法**（先写出）：递归传 `(lower, upper)`，左子树 `(lower, root->val)`，右子树 `(root->val, upper)`
- **中序法**（练习加深）：迭代中序遍历，严格递增即合法
### 🔑 关键代码片段 — 范围法
```cpp
bool dfs(TreeNode* root, long long lower, long long upper) {
    if (!root) return true;
    if (root->val <= lower || root->val >= upper) return false;
    return dfs(root->left, lower, root->val)
        && dfs(root->right, root->val, upper);
}
```
### 🔑 关键代码片段 — 中序迭代法
```cpp
TreeNode *cur = root;
stack<TreeNode*> st;
TreeNode *prev = nullptr;
while (!st.empty() || cur) {
    while (cur) { st.push(cur); cur = cur->left; }
    cur = st.top(); st.pop();
    if (prev && cur->val <= prev->val) return false;
    prev = cur;
    cur = cur->right;
}
```
### 📚 对比原解法
⚠️ **写得很艰难** — 范围法写出来了，中序法需要提示模板后才完成
### ⚠️ 易错点 & 反思
- `LLONG_MIN/MAX` vs `nullptr` 判断第一个节点，后者更优雅
- 中序迭代模板："一路向左 → 出栈处理 → 转向右"
- 两种方法互通：范围法适合递归思路，中序法适合找第k小/验证BST通用

## [Medium] Construct Binary Tree from Preorder and Inorder (105)
### 🎯 核心思路
"前序定根，中序分左右"。前序第一个是根，在中序中找到根的位置，左边是左子树，右边是右子树。用 `in_map` 对中序建立值→下标映射，O(1) 定位。
### 🔑 关键代码片段
```cpp
unordered_map<int, int> in_map;
for (int i = 0; i < inorder.size(); i++) in_map[inorder[i]] = i;

int pre_idx = 0;
TreeNode* build(vector<int>& preorder, int in_left, int in_right,
                int &pre_idx, unordered_map<int,int>& in_map) {
    if (in_left > in_right) return nullptr;
    
    int root_val = preorder[pre_idx++];
    TreeNode* root = new TreeNode(root_val);
    int mid = in_map[root_val];
    
    root->left  = build(preorder, in_left, mid - 1, pre_idx, in_map);
    root->right = build(preorder, mid + 1, in_right, pre_idx, in_map);
    return root;
}
```
### 📚 对比原解法
⚠️ **没独立写出来** — 需要详细解释后才理解原理（看了原文件）
### 🔍 关键角色详解

**`unordered_map<int, int> in_map` — 中序值→下标映射**
```cpp
inorder = [9, 3, 15, 20, 7]
//          0  1   2   3  4
// in_map[9]=0, in_map[3]=1, in_map[15]=2, ...
// 前序取根 3 → in_map[3]=1 → 根在中序位置1
// 左子树 = [0,0]=[9], 右子树 = [2,4]=[15,20,7]
// 类似书本的索引页：告诉我某个值在第几页
```

**`in_left / in_right` — 当前子树在中序数组中的范围**
```
递归层次         in_left  in_right  中序范围
build(全局)      0        4         [9,3,15,20,7]
  建 3 的左子树  0        0         [9]
  建 3 的右子树  2        4         [15,20,7]
    建 20 的左   2        2         [15]
    建 20 的右   4        4         [7]
→ in_left > in_right 时返回 nullptr（没有元素了）
```

**两者协作流程**（以 preorder=[3,9,20,15,7] 为例）：
1. `pre_idx=0` → 取 `3`，查 `in_map[3]=1` → 左范围 [0,0]，右范围 [2,4]
2. `pre_idx=1` → 取 `9`，查 `in_map[9]=0` → 左穷，右穷 → 叶子 ✅
3. `pre_idx=2` → 取 `20`，查 `in_map[20]=3` → 左 [2,2]，右 [4,4]
4. ...

类比：`in_map` 是索引页（查词在第几页），`in_left/in_right` 是当前阅读的页码范围（超出范围不看）。
- `pre_idx` **必须传引用**（`int &`），所有递归层共享全局递增
- **必须先建左子树再建右子树**（前序顺序决定的）
- 空判断用 `in_left > in_right`（中序区间为空），不用检查 preorder 空
- **一句话记**："前序定根 → 中序分左右 → pre_idx 全局引用 → 先左后右"

## [Medium] Lowest Common Ancestor of a Binary Tree (236)
### 🎯 核心思路
后序遍历，自底向上。遇到 p 或 q 就返回，左右子树各返回一个结果，合并判断：
- 左空→返回右，右空→返回左，都非空→返回 root
### 🔑 关键代码片段
```cpp
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root == p || root == q) return root;
    TreeNode* left = lowestCommonAncestor(root->left, p, q);
    TreeNode* right = lowestCommonAncestor(root->right, p, q);
    if (!left) return right;
    if (!right) return left;
    return root;
}
```
### 📚 对比原解法
**一致** ✅ — 一次过，5 行代码完全正确
### ⚠️ 易错点 & 反思
- 终止条件 3 合 1：`!root || root==p || root==q`，遇目标即返回
- 后序保证从底向上：左右结果都回来后才做判断
- **口诀**："搜到目标就返回，只在左右各找到一个时才是 LCA"
- 与 235（BST 版 LCA）的区别：BST 版用值大小三向比较可以剪枝单路递归；普通树只能全量搜索
### 🔍 三个例子走一遍

**例1：p=5, q=4（p 是 q 的祖先）**
```
dfs(3)
├─ dfs(5): 5==p → return 5  ← 直接返回，不往下找4
└─ dfs(1):  左null 右null → return null
left=5, right=null → return 5 ✅
→ LCA=5（因为4是5的子孙，找到5即可）
```

**例2：p=6, q=4（同一子树不同分支）**
```
dfs(5)
├─ dfs(6): 6==p → return 6
└─ dfs(2)
   ├─ dfs(7): null
   └─ dfs(4): 4==q → return 4
   left=null, right=4 → return 4
left=6, right=4 → return 5 ✅
→ LCA=5（6和4分别在5的左右子树）
```

**例3：p=7, q=8（根的两侧）**
```
dfs(3)
├─ dfs(5): ... → return 7
└─ dfs(1): ... → return 8
left=7, right=8 → return 3 ✅
→ LCA=3（一个在左一个在右，根就是LCA）
```

**核心观察**：后序保证了从底向上的返回顺序。一旦某节点左右各找到一个目标，它自己就是 LCA，不再需要往上找了。

## [Medium] Path Sum III (437)
### 🎯 核心思路
前缀和 + 哈希表 + 回溯。从根到当前节点的路径中，`curr_sum - targetSum = need`，need 之前出现过多少次，就有多少条以当前节点结尾的路径。
### 🔑 关键代码片段
```cpp
unordered_map<long long, int> map;
map[0] = 1;
int count = 0;

void dfs(TreeNode* root, long long parent_sum, int targetSum,
         unordered_map<long long, int>& map) {
    if (!root) return;
    long long curr_sum = parent_sum + root->val;
    long long need = curr_sum - targetSum;
    count += map[need];         // 查表，计入路径数
    map[curr_sum]++;            // 当前前缀加入 map
    dfs(root->left, curr_sum, targetSum, map);
    dfs(root->right, curr_sum, targetSum, map);
    map[curr_sum]--;            // 回溯！清掉当前分支的贡献
}
```
### 🔍 完整手走 trace（`[10,5,-3,3,2,null,11,3,-2,null,1]`, target=8）
```
map = {0:1}

dfs(10):   curr=10, need=2,  count=0, map[10]=1
├─ dfs(5): curr=15, need=7,  count=0, map[15]=1
│  ├─ dfs(3左):  curr=18, need=10, map[10]=1→count=1🎉, map[18]=1
│  │  ├─ dfs(3左左): curr=21, need=13, count=0, map[21]=1 → 回溯map[21]--
│  │  └─ dfs(-2):   curr=16, need=8,  count=0, map[16]=1 → 回溯map[16]--
│  │  ← map[18]--   （回溯，清掉3左的贡献）
│  └─ dfs(2):   curr=17, need=9,  count=0, map[17]=1
│     └─ dfs(1): curr=18, need=10, map[10]=1→count=2🎉, map[18]=1 → 回溯map[18]--
│     ← map[17]--   （回溯，清掉2的贡献）
│  ← map[15]--      （回溯，清掉5的贡献！此时map={0:1, 10:1}）
│
└─ dfs(-3):  curr=7,  need=-1, count=0, map[7]=1  ← map干净！只有{0:1,10:1,7:1}
   └─ dfs(11): curr=18, need=10, map[10]=1→count=3🎉
   ← map[7]--
← map[10]--   → map={0:1}

结果: count=3 ✅
路径: [5,3], [5,2,1], [-3,11]
```
### 📚 对比原解法
⚠️ **先看了原文件，没独立写出来** — 理解前缀和+回溯的核心后写正确了
### ⚠️ 易错点 & 反思
- `map[0]=1` 处理从根节点开始的路径（curr_sum == targetSum 时 need=0）
- `long long` 防溢出（节点值可达 ±1e9，递归深度可达 1e5）
- **回溯是核心**：`map[curr_sum]--` 保证不同分支互不干扰
- 走到右子树时，左子树的 map 条目一定要全清零！
- **一句记**："当前前缀减目标，查表计路再加表，回溯清账不干扰"

## [Easy] Diameter of Binary Tree (543)
### 🎯 核心思路
后序遍历算子树高度，每个节点用 `左高+右高` 更新全局直径。
### 🔑 关键代码片段
```cpp
int ans = 0;
int dfs(TreeNode* root) {
    if (!root) return 0;
    int left = dfs(root->left);
    int right = dfs(root->right);
    ans = max(ans, left + right);  // 经过当前节点的最长路径
    return max(left, right) + 1;   // 返回给父节点的高度
}
int diameterOfBinaryTree(TreeNode* root) {
    dfs(root);
    return ans;
}
```
### 📚 对比原解法
**一致** ✅ — 一次过
### ⚠️ 易错点 & 反思
- 直径是 **边数** 不是节点数（`左高+右高` 即边数）
- 这条路径**不一定经过根**，所以每个节点都要更新 ans
- 配套题：687（最长同值路径），多了值相等的判断和清零
