# 📝 2026-05-14 学习记录

## [Medium] Path Sum III (437)

### 🎯 核心思路
**前缀和 + 哈希表 + 回溯**。一次 DFS 遍历树，用 `unordered_map<long long, int>` 记录从根到当前节点路径上各前缀和出现的次数。对于当前节点，查表找 `curr_sum - targetSum` 出现的次数，累加到结果中。递归返回前回溯，移除当前前缀和，确保只统计当前路径上的前缀和。

### 🔑 关键代码片段
```cpp
class Solution {
    int count = 0;
public:
    int pathSum(TreeNode* root, int targetSum) {
        unordered_map<long long, int> map;
        map[0] = 1;
        dfs(root, 0, targetSum, map);
        return count;
    }
    void dfs(TreeNode* root, long long parent_sum, int targetSum,
             unordered_map<long long, int>& map) {
        if (!root) return;
        long long curr_sum = parent_sum + root->val;
        long long need = curr_sum - targetSum;
        count += map[need];
        map[curr_sum]++;
        dfs(root->left,  curr_sum, targetSum, map);
        dfs(root->right, curr_sum, targetSum, map);
        map[curr_sum]--;
    }
};
```

### 📚 学到的知识点
- **树上前缀和**：从根到当前节点路径上累加，与数组前缀和本质相同，但需要回溯
- **逆向思维**：不是"从当前节点往下找=targetSum"，而是"之前有没有前缀和 = curr_sum - targetSum"
- **`map[0] = 1`**：处理根到当前节点整条路径和正好等于 targetSum 的情况
- **回溯的必要性**：不同路径分支互不影响，必须用 `map[curr_sum]--` 撤销当前分支的贡献
- **long long 防溢出**：节点值范围大，int 可能溢出；且 map key 类型也要用 long long，否则截断可能导致误命中 map[0]

### ⚠️ 易错点 & 反思
- ❌ `curr_sum` 和 `need` 忘记用 `long long` → 203 测试用例 int 溢出
- ❌ map key 类型仍为 `int` → `curr_sum` 截断后撞到 `map[0]`，误加计数
- ❌ 忘记 `map[0] = 1` → 根到当前节点整条路径和为 targetSum 的情况漏统计
- ❌ 忘记回溯 `map[curr_sum]--` → 不同路径分支间的前缀和互相干扰
- 💡 **路径三部曲**：112（有没有）→ 113（列路径/根到叶子）→ 437（计数/任意路径）。同个问题逐步升级

## 💬 补充讨论：如果要列路径怎么做？

437 要求**计数**，前缀和查表刚好满足（O(n)）。但如果要**列出所有路径**，有两种做法：

### 方向一：双重 DFS（暴力法，O(n²)）

改造 [113] 的 "根→叶子" 为 "任意起点→任意终点"：

```cpp
// 内层：以某个节点为起点，向下搜所有路径
void dfs_from(TreeNode* node, long long sum, vector<int>& path, vector<vector<int>>& res) {
    if (!node) return;
    path.push_back(node->val);
    sum -= node->val;
    if (sum == 0) res.push_back(path);  // 任何节点都检查，不限于叶子
    dfs_from(node->left,  sum, path, res);
    dfs_from(node->right, sum, path, res);
    path.pop_back();
}

// 外层：遍历每个节点，作为起点
void traverse(TreeNode* root, int targetSum, vector<vector<int>>& res) {
    if (!root) return;
    vector<int> path;
    dfs_from(root, targetSum, path, res);  // 以 root 为起点
    traverse(root->left,  targetSum, res); // 换左孩子为起点
    traverse(root->right, targetSum, res); // 换右孩子为起点
}
```

| 做法 | 时间复杂度 | 空间复杂度 | 能列路径？ |
|:---:|:---------:|:---------:|:--------:|
| 双重 DFS | O(n²) | O(n) | ✅ |
| 前缀和+哈希 | O(n) | O(n) | ❌仅计数 |

### 方向二：路径向量 + 前缀和（进阶，O(n)）

- `unordered_map<long long, vector<int>>` 存 `前缀和 → 出现时的路径长度列表`
- 同时维护 `vector<int> path` 记录当前遍历路径
- 找到 `curr_sum - targetSum` 时，用路径长度索引在 `path` 中切片，得到具体路径
- 回溯时同时 pop path 和移除 map 中的长度记录

理论上 O(n) 可列路径，但实现较复杂，通常面试用方向一就够了。
