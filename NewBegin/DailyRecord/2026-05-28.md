## [Medium] Permutations (46)
### 🎯 核心思路
回溯 = DFS + 状态重置。用 used 数组标记已选数字，递归生成所有排列。

### 🔑 关键代码片段
```cpp
vector<vector<int>> res;
vector<bool> used(nums.size(), false);
vector<int> path;
backtrack(nums, used, path, res);

void backtrack(vector<int>& nums, vector<bool>& used, vector<int>& path, vector<vector<int>>& res) {
    if (path.size() == nums.size()) {
        res.push_back(path);
        return;
    }
    for (int i = 0; i < nums.size(); i++) {
        if (used[i]) continue;
        used[i] = true;
        path.push_back(nums[i]);
        backtrack(nums, used, path, res);
        used[i] = false;
        path.pop_back();
    }
}
```

### 📚 学到的知识点
1. **回溯框架**：做选择 → 递归下一层 → 撤销选择
2. **used 数组**：标记哪些元素已被使用，避免重复选取
3. **和 DFS 的区别**：图 DFS 标记后不用撤销，回溯必须撤销

### ⚠️ 易错点 & 反思
- ❌ `vector<bool> used;` 没初始化大小 → 运行时越界崩溃
- ✅ `vector<bool> used(nums.size(), false);`
- 撤销顺序和做选择顺序相反：先 pop 再改 used

## [Medium] Subsets (78)
### 🎯 核心思路
回溯 + start 索引。每次进入 backtrack 先记录当前 path，从 start 开始遍历选元素，下一层从 i+1 开始。

### 🔑 关键代码片段
```cpp
void backtrack(vector<int>& nums, int start, vector<int>& path, vector<vector<int>>& res) {
    res.push_back(path);
    for (int i = start; i < nums.size(); i++) {
        path.push_back(nums[i]);
        backtrack(nums, i + 1, path, res);
        path.pop_back();
    }
}
```

### 📚 学到的知识点
1. **start 索引**：控制不走回头路，保证子集不重复
2. **每层都记录**：和排列不同，子集是所有中间状态，包括空集
3. **i+1 不是 start+1**：选完 nums[i] 后下一层从 i+1 开始

### ⚠️ 易错点 & 反思
- ❌ `res.push_back(path)` 放在 for 循环里 → 空集 [ ] 不会入结果
- ❌ `backtrack(nums, start+1, ...)` → 同一元素在当前层被重复选取
- ✅ 先进 backtrack 就 push，再 for 循环做选择

## [Medium] Combination Sum (39)
### 🎯 核心思路
回溯 + start 索引，元素可重复选（下一层传 i 不 +1）。先排序后用 sum 剪枝，sum==target 时记录。

### 🔑 关键代码片段
```cpp
sort(candidates.begin(), candidates.end());
void backtrack(vector<int>& candidates, int start, int sum, vector<int>& path, vector<vector<int>>& res, int target) {
    if (sum == target) { res.push_back(path); return; }
    for (int i = start; i < candidates.size(); i++) {
        if (sum + candidates[i] > target) break;  // 排序后剪枝
        path.push_back(candidates[i]);
        backtrack(candidates, i, sum + candidates[i], path, res, target);
        path.pop_back();
    }
}
```

### 📚 学到的知识点
1. **可重复选**：下一层传 i 而不是 i+1，同一个元素可以在后续递归中再次被选
2. **排序剪枝**：排序后一旦 sum+candidates[i] > target，后面的更大，直接 break
3. **sum 传值**：递归传 `sum + candidates[i]` 不用恢复，path 传引用需要 pop_back

### ⚠️ 易错点 & 反思
- ❌ `backtrack(candidates, start, ...)` → 产生 [3,2,2] 重复组合
- ❌ `sum += candidates[i]` 不恢复 → 下一轮 sum 累加错误
- ❌ 没排序就 break → 后面可能有更小的数能凑上
- ✅ 排序 + 传 i + sum 传值
