## [Medium] Number of Islands (200)
### 🎯 核心思路
网格 DFS（水淹法）：遍历整个网格，遇到 `'1'` 就计数 + DFS 淹没整块岛屿（标记为 `'0'`）。
### 🔑 关键代码片段
```cpp
int dirs[4][2] = {{1,0},{-1,0},{0,1},{0,-1}};

void dfs(vector<vector<char>>& grid, int i, int j) {
    if (i < 0 || i >= grid.size() || j < 0 || j >= grid[0].size() || grid[i][j] == '0')
        return;
    grid[i][j] = '0';
    for (auto& d : dirs) dfs(grid, i + d[0], j + d[1]);
}

int numIslands(vector<vector<char>>& grid) {
    int res = 0;
    for (int i = 0; i < grid.size(); i++)
        for (int j = 0; j < grid[0].size(); j++)
            if (grid[i][j] == '1') { res++; dfs(grid, i, j); }
    return res;
}
```
### 📚 对比原解法
**一致** ✅ — 思路完全正确，方向数组写法更规范了
### ⚠️ 易错点 & 反思
- 内层循环条件 `i < grid[0].size()` 写成了 `i` 而不是 `j`，导致死循环 💥
- 网格题建议习惯性先 `if (grid.empty() || grid[0].empty()) return 0;`
- 边界条件合并写在一个 if 里比分开写更简洁
- BFS 版本可用队列避免递归爆栈，但递归版更简洁
- 与 695 (Max Area) 的区别：695 的 DFS 需要返回面积，200 只需淹没标记

## [Medium] Course Schedule (207)
### 🎯 核心思路
三色 DFS 检测有向图环：0=未访问（白），1=正在访问（灰），2=已访问（黑）。遇到灰色即发现环。
### 🔑 关键代码片段
```cpp
vector<vector<int>> graph(numCourses);
for (auto& p : prerequisites) graph[p[1]].push_back(p[0]);

vector<int> status(numCourses, 0);  // 0=白, 1=灰, 2=黑

bool dfs(const vector<vector<int>>& graph, vector<int>& status, int node) {
    if (status[node] == 1) return false;  // 灰 → 环
    if (status[node] == 2) return true;   // 黑 → 已确认无环
    status[node] = 1;
    for (int neighbor : graph[node])
        if (!dfs(graph, status, neighbor)) return false;
    status[node] = 2;
    return true;
}
```
### 📚 对比原解法
**一致** ✅ — 三色 DFS 写得完全正确，建图方式一致
### 🔍 三色法可视化（示例2)
```
dfs(0) → status[0]=1 (灰)
  └─ dfs(1) → status[1]=1 (灰)
       └─ dfs(0) → status[0]==1 → 遇到灰！环！💥
```
### ⚠️ 易错点 & 反思
- 三色法的巧妙：2 种状态不够（bool visited 分不清"正在访问"和"已访问完"），3 种才能检测有向环
- 遇到 1（灰）= 当前递归栈上这个节点正在处理 = 有环
- 遇到 2（黑）= 之前确认过无环，直接剪枝
- 建图方向：`[ai, bi]` → `bi → ai`（先修课指向后修课），不要搞反
- 外层循环用 `!status[i] && !dfs()` 短路：已黑的跳过，没黑的遇到环才返回 false
- 210 题只多一步：在 `status[node]=2` 时收集到答案数组即可得拓扑顺序

## [Medium] Surrounded Regions (130)
### 🎯 核心思路
逆向思维：从边界 O 出发 DFS，标记连到边界的 O 为 `'T'`（临时标记），然后遍历全图：`'O'→'X'`（被包围的）、`'T'→'O'`（还原连边界的）。
### 🔑 关键代码片段
```cpp
void solve(vector<vector<char>>& board) {
    if (board.empty()) return;
    int m = board.size(), n = board[0].size();
    // Step 1: 从边界 O 出发 DFS，标记为 'T'
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++)
            if ((i==0 || j==0 || i==m-1 || j==n-1) && board[i][j]=='O')
                dfs(board, i, j);
    // Step 2: 遍历全图，O→X（被包围），T→O（还原）
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++)
            if (board[i][j] == 'O') board[i][j] = 'X';
            else if (board[i][j] == 'T') board[i][j] = 'O';
}
void dfs(vector<vector<char>>& board, int i, int j) {
    if (i<0 || j<0 || i>=board.size() || j>=board[0].size() || board[i][j]!='O')
        return;
    board[i][j] = 'T';
    for (auto& d : dir) dfs(board, i + d[0], j + d[1]);
}
```
### 📚 对比原解法
**一致** ✅ — 逆向思维完全正确，三步骤流程清晰
### ⚠️ 易错点 & 反思
- 🐛 **变量遮蔽（shadow）**：DFS 中 `for (int i = 0; i < 4; i++)` 的 `i` 覆盖了参数 `i`，导致递归坐标全错！→ 循环变量用 `d`/`k` 而非 `i`
- **逆向思维通用模式**："不该改的"先标记 → 剩下的全改 → 还原标记。类似 417 (Pacific Atlantic) 的"反向 DFS→取交集"
- 防御检查：`board.empty()` 能防空输入，但 `board[0].empty()` 也需要考虑（空列情况）

## [Medium] Rotting Oranges (994)
### 🎯 核心思路
多源 BFS + 分层计时：所有烂橘子同时入队，按层扩散，每层 = 1 分钟。
### 🔑 关键代码片段
```cpp
queue<pair<int, int>> q;
int fresh = 0, minutes = 0;
// 所有烂橘子入队，统计新鲜橘子数
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        if (grid[i][j] == 2) q.push({i, j});
        else if (grid[i][j] == 1) fresh++;

while (!q.empty() && fresh) {
    int size = q.size();
    while (size--) {
        auto [i, j] = q.front(); q.pop();
        for (auto& d : dir) {
            int ni = i + d[0], nj = j + d[1];
            if (ni>=0 && ni<m && nj>=0 && nj<n && grid[ni][nj]==1) {
                grid[ni][nj] = 2; fresh--;
                q.push({ni, nj});
            }
        }
    }
    minutes++;
}
return fresh == 0 ? minutes : -1;
```
### 📚 对比原解法
**一致** ✅ — 多源 BFS 模板完全正确，分层计时写得好
### ⚠️ 易错点 & 反思
- `pair` 用法：`queue<pair<int,int>>` + `q.push({i,j})` + `q.front().first/second`
- C++17 结构化绑定 `auto [i, j] = q.front()` 可让代码更清晰
- 方向数组 vs 4 个 if：功能等价，方向数组更 DRY，4 个 if 更直观
- **关键优化**：用 `fresh` 变量计数，避免最后全遍历检查 O(m×n)，同时可以提前结束（`while(... && fresh)`）
- 与 542 的关系：994 的分钟数 = 542 的距离，都是多源 BFS 逐层计数

## [Medium] 01 Matrix (542)
### 🎯 核心思路
多源 BFS：所有 0 同时入队，距离初始为 0；其余格子初始 `INT_MAX`（未访问）。BFS 首次访问即更新为最短距离。
### 🔑 关键代码片段
```cpp
vector<vector<int>> res(m, vector<int>(n, INT_MAX));
queue<pair<int, int>> q;
for (int i = 0; i < m; i++)
    for (int j = 0; j < n; j++)
        if (mat[i][j] == 0) { res[i][j] = 0; q.push({i, j}); }

while (!q.empty()) {
    auto [x, y] = q.front(); q.pop();
    for (auto& d : dir) {
        int nx = x + d[0], ny = y + d[1];
        if (nx >= 0 && nx < m && ny >= 0 && ny < n && res[nx][ny] == INT_MAX) {
            res[nx][ny] = res[x][y] + 1;
            q.push({nx, ny});
        }
    }
}
```
### 📚 对比原解法
**一致** ✅ — 多源 BFS 完全正确，INT_MAX 做未访问标记很巧妙
### ⚠️ 易错点 & 反思
- 与 994 的区别：542 **不需要分层**（`size` 循环），因为每个格子单独存距离而非统计层数
- `INT_MAX` 作为未访问标记：比 bool visited 省一个数组，+1 也不会溢出（格数 ≤ 10⁴）
- 结构化绑定 `auto [x, y]` 让 pair 操作更可读
- **关键理解**：BFS 第一次到达的距离 = 最短距离，这是 BFS 的天然性质
