## [Medium] Word Search (79)
### 🎯 核心思路
网格回溯：在网格中找 word[0] 启动 DFS，边走边匹配字符。匹配则标记→递归→恢复，不匹配或越界立即返回。

### 🔑 关键代码片段
```cpp
bool dfs(vector<vector<char>>& board, string& word, int k, int i, int j) {
    int m = board.size(), n = board[0].size();
    if (i < 0 || i >= m || j < 0 || j >= n) return false;
    if (board[i][j] != word[k]) return false;
    if (k == word.size() - 1) return true;

    char temp = board[i][j];
    board[i][j] = '#';
    for (auto& d : dirs)
        if (dfs(board, word, k + 1, i + d[0], j + d[1]))
            return true;
    board[i][j] = temp;
    return false;
}
```

### 📚 学到的知识点
1. **原数组标记**：`board[i][j] = '#'` 代替 visited 数组，省空间
2. **标记后必须恢复**：回溯的核心——一条路径失败后要恢复现场
3. **传引用**：`string& word` 避免递归拷贝

### ⚠️ 易错点 & 反思
- ❌ **变量名覆盖**：`for (int i...)` 覆盖参数 i，坐标全乱
- ❌ **缺少边界检查**：越界访问 board[-1][j] 崩溃
- ❌ **string 传值**：每次递归拷贝字符串，超时
- ❌ **缺少终止条件**：k==len-1 时要 return true

### 📌 回溯 / DFS 模板总结

**回溯三模板：**
```
46 排列    → used 数组 + 叶子记录
78 子集    → start 索引 + 每层记录

79 单词搜索 → 网格回溯 = 标记 + 递归 + 恢复
```

**DFS 在网格上的两种用法：**
```
200 岛屿数量 → 标记后不恢复（水淹法）
79 单词搜索 → 标记后必须恢复（回溯法）
```

**所有回溯问题的统一框架：**
```
void backtrack(状态参数) {
    if (满足条件) { 记录结果; return; }
    for (选择 in 所有可能的下一步) {
        if (选择无效) continue;        // 剪枝
        做选择（标记/加入路径）;
        backtrack(更新后的状态);
        撤销选择（恢复标记/弹出路径）;
    }
}
```

## [Medium] Generate Parentheses (22)
### 🎯 核心思路
回溯 + 左右括号计数约束。left 记录剩余左括号，right 记录剩余右括号。放 ( 条件 left>0，放 ) 条件 right>left。

### 🔑 关键代码片段
```cpp
void backtrack(vector<string>& res, string& s, int left, int right) {
    if (left == 0 && right == 0) {
        res.push_back(s);
        return;
    }
    if (left > 0) {
        s.push_back('(');
        backtrack(res, s, left - 1, right);
        s.pop_back();
    }
    if (right > left) {
        s.push_back(')');
        backtrack(res, s, left, right - 1);
        s.pop_back();
    }
}
```

### 📚 学到的知识点
1. **约束式回溯**：用 left/right 剩余数量控制分支，不需要 used/start
2. **有效的核心条件**：任何时候右括号数量不能超过左括号 → `right > left` 才能放 )

### ⚠️ 易错点 & 反思
- `right > left` 是能放的条件，不是 `right > 0`——否则会产生非法组合
- 用剩余数量比用已用数量更直观

## [Medium] Letter Combinations of a Phone Number (17)
### 🎯 核心思路
映射 + 回溯。用 string 数组存映射（下标 0 对应数字 2），遍历当前数字对应的所有字母，递归处理下一个数字。

### 🔑 关键代码片段
```cpp
const string mapping[8] = {"abc","def","ghi","jkl","mno","pqrs","tuv","wxyz"};
//                         2     3     4     5     6     7      8     9
void backtrack(vector<string>& res, string& s, int k, const string mapping[], const string& digits) {
    if (k == digits.size()) { res.push_back(s); return; }
    for (char c : mapping[digits[k] - '2']) {
        s.push_back(c);
        backtrack(res, s, k + 1, mapping, digits);
        s.pop_back();
    }
}
```

### 📚 学到的知识点
1. **string 数组代替哈希表**：`mapping[digits[k]-'2']` 比 unordered_map 更简洁
2. **和 22 一样不需要 used/start**：每一步从映射中取字母，按顺序推进

### ⚠️ 易错点 & 反思
- ❌ `digits=""` 时返回 `[""]` 而不是 `[]` → 需要 `if (digits.empty()) return {};`
- `const string& digits` 传引用省拷贝
