# 📚 2026-06-02 周复习 Day 2 — 栈 + 二分搜索

## [Medium] Daily Temperatures (739)
### 🎯 核心思路
单调递减栈存下标。遇到更高温度时弹栈，计算天数差 `i - top`。
### 🔑 关键代码片段
```cpp
vector<int> res(temperatures.size(), 0);
stack<int> st;
for (int i = 0; i < temperatures.size(); i++) {
    if (st.empty()) {
        st.push(i);
    } else {
        while (!st.empty() && temperatures[st.top()] < temperatures[i]) {
            int top = st.top(); st.pop();
            res[top] = i - top;
        }
        st.push(i);
    }
}
return res;
```
### 📚 对比原解法
**一致**——写法完全相同 ✅
### ⚠️ 易错点 & 反思
- `if (st.empty())` 分支可省略，`while` 条件已处理空栈情况

## [Medium] Next Greater Element II (503)
### 🎯 核心思路
循环数组处理三要素：`n*2` 遍历 → `i%n` 取模 → `if(i<n)` 只入栈一次。
### 🔑 关键代码片段
```cpp
int n = nums.size();
vector<int> res(n, -1);
stack<int> st;
for (int i = 0; i < n * 2; i++) {
    while (!st.empty() && nums[st.top()] < nums[i % n]) {
        int top = st.top(); st.pop();
        res[top] = nums[i % n];
    }
    if (i < n) st.push(i);
}
return res;
```
### 📚 对比原解法
**一致** ✅
### ⚠️ 易错点 & 反思
- ❌ 第一次忘了 `i*2`，导致最后一位没处理到 → 循环数组必写 `n*2` + `i%n` + `if(i<n)` 三连
- 核心口诀："两倍遍历，取模下标，一圈入栈"

## [Hard] Trapping Rain Water (42)
### 🎯 核心思路
单调递增栈按**层**计算积水。弹栈时：`宽度 × 层高 = (右-左-1) × (min(左边高,右边高) - 底部高)`。
### 🔑 关键代码片段
```cpp
int res = 0;
stack<int> st;
for (int i = 0; i < n; i++) {
    while (!st.empty() && height[st.top()] < height[i]) {
        int top = st.top();  // 底部
        st.pop();
        if (st.empty()) break;  // 无左边界 → 无法接水
        res += (i - st.top() - 1) * (min(height[st.top()], height[i]) - height[top]);
    }
    st.push(i);
}
```
### 📚 对比原解法
**一致** ✅ — 不过第一次写时思路跑偏：
- ❌ 用 for 循环逐列累加（按列加水）
- ❌ 把底部当左边界用了
- ✅ 修正后理解：弹出来的是**底部**，剩在栈顶是**左边界**，i 是**右边界**，一次乘法算一层
### ⚠️ 易错点 & 反思
- **三个角色不能混淆**：`st.top()` 弹完是左墙，弹出去的是池底，`i` 是右墙
- **公式一句记**：`(i - st.top() - 1) * (min(height[st.top()], height[i]) - height[top])`
- **为什么是单调递增栈**：只有遇到比栈顶高的右墙时，前面才能形成凹槽接水

## [Medium] Find First and Last Position (34)
### 🎯 核心思路
lower_bound 两次：找 ≥target（左边界）和 ≥target+1（右边界+1），不存在时两值相等。
### 🔑 关键代码片段
```cpp
int lower_bound(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target) right = mid - 1;
        else                     left = mid + 1;
    }
    return left;
}
int leftbound = lower_bound(nums, target);
int rightbound = lower_bound(nums, target + 1);
if (leftbound == rightbound) return {-1, -1};
return {leftbound, rightbound - 1};
```
### 📚 对比原解法
**一致** ✅ — lower_bound 模板一次写对
### ⚠️ 易错点 & 反思
- `right` 初始 `nums.size()-1` 不是 `nums.size()`
- `nums[mid] >= target` 等号不能漏
- 左边界 = lower_bound(target)，右边界 = lower_bound(target+1) - 1

## [Medium] Search in Rotated Sorted Array (33)
### 🎯 核心思路
先判 mid 在左段还是右段（`nums[0] <= nums[mid]`），再在段内二分。
### 🔑 关键代码片段
```cpp
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] == target) return mid;
    if (nums[0] <= nums[mid]) {  // mid 在左段
        if (nums[0] <= target && target < nums[mid])
            right = mid - 1;
        else left = mid + 1;
    } else {                     // mid 在右段
        if (nums[0] > target && nums[mid] < target)
            left = mid + 1;
        else right = mid - 1;
    }
}
```
### 📚 对比原解法
⚠️ **第一次写法有 bug** — 左/右段的 else 分支逻辑反了，导致 target 在另一个段时被剪掉
- ✅ 修正后一次过
### ⚠️ 易错点 & 反思
- `>=` 不能写成 `>`（mid=0 时 nums[0]==nums[0] 也算左段）
- 口诀："左段左小，右段右大" — target 在自己段且在收缩方向才收，否则往外走
- 每个段的 else 分支必须覆盖 `target 在另一段` 的情况

## [Medium] Koko Eating Bananas (875)
### 🎯 核心思路
**二分答案**：在值域 `[1, max(piles)]` 上二分速度 k，判断 `canEat(k) <= h`。
### 🔑 关键代码片段
```cpp
int left = 1, right = *max_element(piles.begin(), piles.end());
while (left < right) {
    int mid = left + (right - left) / 2;
    if (canEat(mid, piles, h)) right = mid;
    else                       left = mid + 1;
}
return left;

bool canEat(int k, vector<int>& piles, int h) {
    int count = 0;
    for (int p : piles) {
        count += (p + k - 1) / k;  // 向上取整
        if (count > h) return false;
    }
    return true;
}
```
### 📚 对比原解法
⚠️ **第一次写错两个地方**：
- ❌ 在数组下标上二分（`left=0, right=piles.size()-1`）→ 应该在值域上二分
- ❌ while 里忘了更新 left/right → 死循环
- ✅ 修正后一次过
### ⚠️ 易错点 & 反思
- **二分答案 ≠ 二分数组**：答案区间是 `[1, max]`，不是 `[0, n-1]`
- `while (left < right)` + `right = mid` / `left = mid + 1` 是求最小可行解的模板
- 向上取整：`(p + k - 1) / k`
