## [Medium] Construct Binary Tree from Preorder and Inorder Traversal (105)

### 🎯 核心思路
**前序定根，中序分左右**。前序第一个元素是根，在中序中找到根的位置 `mid`，左边是左子树，右边是右子树。递归构建。

### 🔑 关键代码片段
```cpp
TreeNode* buildTree(vector<int>& preorder, vector<int>& inorder) {
    unordered_map<int, int> in_map;
    for (int i = 0; i < inorder.size(); i++)
        in_map[inorder[i]] = i;
    int pre_idx = 0;
    return build(preorder, 0, inorder.size() - 1, pre_idx, in_map);
}
TreeNode* build(vector<int>& preorder, int in_left, int in_right,
                int& pre_idx, unordered_map<int, int>& in_map) {
    if (in_left > in_right) return nullptr;
    int root_val = preorder[pre_idx++];
    TreeNode* root = new TreeNode(root_val);
    int mid = in_map[root_val];
    root->left  = build(preorder, in_left, mid - 1, pre_idx, in_map);
    root->right = build(preorder, mid + 1, in_right, pre_idx, in_map);
    return root;
}
```

### 🧠 细节详解

#### ① `mid` 到底是什么？怎么来的？

```
前序 preorder = [3, 9, 20, 15, 7]
中序 inorder  = [9, 3, 15, 20, 7]
                  0  1   2   3  4
```

`mid` = **根节点在中序数组中的索引位置**。通过哈希表 O(1) 查到：

```
in_map[9]=0, in_map[3]=1, in_map[15]=2, in_map[20]=3, in_map[7]=4

int root_val = preorder[pre_idx];   // 当前根的值，比如第一次是 3
int mid = in_map[root_val];         // mid = in_map[3] = 1
```

得到 `mid = 1` 后，中序数组被切成两半：

```
中序: [9, 3, 15, 20, 7]
         ↑  ↑
      左边  右边
    in_left..mid-1  mid+1..in_right
    = [0..0]        = [2..4]
    = [9]           = [15,20,7]
    = 左子树         = 右子树
```

#### ② 完整递归推演（以 [3,9,20,15,7] / [9,3,15,20,7] 为例）

```
第 1 层: pre_idx=0, in_left=0, in_right=4
  root_val = preorder[0] = 3, pre_idx→1
  mid = in_map[3] = 1
  左子树区间 [0..0] = [9]    右子树区间 [2..4] = [15,20,7]

第 2 层(左): pre_idx=1, in_left=0, in_right=0
  root_val = preorder[1] = 9, pre_idx→2
  mid = in_map[9] = 0
  左子树区间 [0..-1] = nullptr  右子树区间 [1..0] = nullptr
  → 返回节点 9

回到第 1 层, root->left = 9

第 3 层(右): pre_idx=2, in_left=2, in_right=4
  root_val = preorder[2] = 20, pre_idx→3
  mid = in_map[20] = 3
  左子树区间 [2..2] = [15]  右子树区间 [4..4] = [7]

  ...递归构建 15 和 7...
```

#### ③ 为什么 `pre_idx` 必须传引用？

因为前序指针要**全局递增**，每取一个根就后移，不能回溯：

```
preorder = [3, 9, 20, 15, 7]
            0   1   2   3  4

build 顺序: 取3(pre_idx=0) → 取9(pre_idx=1) → ... → 取7(pre_idx=4)

传引用: pre_idx 一路递增，不会回到之前的值  ✅
传值  : 每层递归有自己的 pre_idx，回溯后会重复取值  ❌
```

#### ④ 为什么必须先构建左子树？

前序顺序是 **根→左→右**：

```
preorder = [根, 左子树..., 右子树...]
              3   9    |  20, 15, 7
              ↑       ↑
          pre_idx=0  pre_idx=1
```

先 `root->left` 把左子树节点 `9` 消耗掉，`pre_idx` 指向 `20`，再构建右子树就正好取到右树根 `20`。

### 📚 学到的知识点
- **前序**：第一个是根，后面按左→右排列
- **中序**：根左边是左子树，右边是右子树
- **用 `mid` 切分中序区间**：左子树 `[in_left, mid-1]`，右子树 `[mid+1, in_right]`
- **`pre_idx` 传引用**：全局递增，每取一个根就后移，不能回溯
- **递归顺序**：必须先左后右（前序顺序决定）

### ⚠️ 易错点 & 反思
- ❌ `pre_idx` 忘记传引用 → 递归回溯后重复取值
- ❌ 先构建右子树再构建左子树 → 前序顺序破坏，取错根
- 💡 **口诀**："前序定根，中序分左右；传引用指针，先左再右走"

## [Medium] Path Sum II (113)

### 🎯 核心思路
**DFS + 回溯记录路径**。与 [112] Path Sum 不同，113 不仅要判断有没有，还要把所有路径记录下来。用 `vector<int> path` 追踪当前路径，到达叶子节点且和为目标值时存入结果集，递归返回前 `pop_back()` 回溯。

### 🔑 关键代码片段
```cpp
void dfs(TreeNode* root, int sum, vector<int>& path, vector<vector<int>>& res) {
    if (!root) return;
    path.push_back(root->val);     // ① 路径记录当前节点
    sum -= root->val;              // ② 减去当前值
    if (!root->left && !root->right && sum == 0)   // ③ 叶子 + 符合条件
        res.push_back(path);                        //    存路径
    dfs(root->left,  sum, path, res);               // ④ 递归左
    dfs(root->right, sum, path, res);               // ⑤ 递归右
    path.pop_back();                 // ⑥ 回溯！撤销当前节点，恢复状态
}
```

### 🧠 细节详解

#### ① 为什么必须 `pop_back()`？

DFS 遍历树时，`path` 会沿一条路径向下走。如果没有 `pop_back`，从一个节点**返回**时，`path` 中还带着它：

```
5 → 5,4 → 5,4,11 → 5,4,11,7 → (7 走完) → 5,4,11,7,2 ❌
                                    ↑ 没 pop 7，path 没回退！
```

有 `pop_back` 后：
```
5 → 5,4 → 5,4,11 → 5,4,11,7 → (pop 7) → 5,4,11 → 5,4,11,2 → ...
                                                         ↑ 正确！
```

**核心思想：有加就有减，递归前后状态要一致。**

#### ② 叶子节点的判断

```cpp
if (!root->left && !root->right && sum == 0)
```
- `!root->left && !root->right`：左右都没有孩子，才是叶子
- 只有叶子才检查，非叶子节点即使 sum==0 也不算路径完成

#### ③ 与 [112] Path Sum 对比

| 对比 | [112] Path Sum | [113] Path Sum II |
|:----:|:---------------|:-------------------|
| 返回值 | `bool` 有没有 | `vector<vector<int>>` 所有路径 |
| 路径记录 | 不需要 | `vector<int> path` 记录当前路径 |
| 回溯 | 不需要 | 需要 `pop_back()` |
| 提前返回 | 找到就返回 true | 必须遍历所有可能路径 |

### ⚠️ 易错点 & 反思
- ❌ 忘记 `pop_back()` → path 越走越长
- ❌ 非叶子节点也检查路径和 → 要求的是**根到叶子**的完整路径
- ❌ `res.push_back(path)` 不用 `path` 副本？→ `res` 存的是 `path` 的**拷贝**，后续的 pop 不影响已存的路径
- 💡 **回溯模板**：push → 递归 → pop，三步固定搭配
