## [Medium] Number of Islands (200)

### 🎯 核心思路
**DFS + 水淹法**。遍历网格，遇到 `'1'` 就计数 +1，然后从该格子开始 DFS 把整个岛屿淹掉（`'1'`→`'0'`）。四方向搜索（上、下、左、右），越界或遇到 `'0'` 就返回。

### 🔑 关键代码片段
```cpp
class Solution {
public:
    int numIslands(vector<vector<char>>& grid) {
        int dirs[4][2] = {{1,0}, {-1,0}, {0,1}, {0,-1}};
        int m = grid.size(), n = grid[0].size();
        int res = 0;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                if (grid[i][j] == '1') {
                    res++;
                    dfs(grid, i, j, dirs);
                }
        return res;
    }
    void dfs(vector<vector<char>>& grid, int i, int j, int dirs[4][2]) {
        int m = grid.size(), n = grid[0].size();
        if (i < 0 || i >= m || j < 0 || j >= n || grid[i][j] == '0')
            return;
        grid[i][j] = '0';
        for (int k = 0; k < 4; k++)
            dfs(grid, i + dirs[k][0], j + dirs[k][1], dirs);
    }
};
```

### 📚 学到的知识点
- **网格 DFS vs 树 DFS**：树有 left/right 两个方向，网格有 4 个方向
- **水淹法（标记已访问）**：直接修改原数组 `'1'→'0'`，省去 visited 数组
- **外层双循环扫描**：遍历每个格子，遇到陆地就计数并淹掉整个岛屿

### ⚠️ 易错点 & 反思
- ❌ 搜索只考虑右和下方 → 会漏掉上方和左方的陆地，必须四方向搜索
- ❌ 忘记边界检查 `i < 0 || i >= m || j < 0 || j >= n`
- ❌ 字符比较用 `grid[i][j] == 1` → 应该是 `== '1'`
- 💡 水淹法本质是"每次发现新大陆就淹掉它"，相当于每个岛屿只被统计一次

- [463] Island Perimeter — 计算岛屿周长
- [130] Surrounded Regions — 从边界反向 DFS

## [Medium] Clone Graph (133)

### 🎯 核心思路
**DFS + 哈希表深拷贝**。`unordered_map<Node*, Node*>` 存原节点→克隆节点的映射。DFS 递归遍历：已克隆则直接返回；否则创建新节点，先存入 map 再递归克隆邻居，最后返回克隆节点。先存再递归是关键，防止环导致死循环。

### 🔑 关键代码片段
```cpp
// DFS 版
Node* cloneGraph(Node* node) {
    if (!node) return nullptr;
    unordered_map<Node*, Node*> visited;
    return dfs(node, visited);
}
Node* dfs(Node* node, unordered_map<Node*, Node*>& visited) {
    if (visited.find(node) != visited.end())
        return visited[node];
    Node* clone = new Node(node->val);
    visited[node] = clone;
    for (auto neighbor : node->neighbors)
        clone->neighbors.push_back(dfs(neighbor, visited));
    return clone;
}

// BFS 版
Node* cloneGraph(Node* node) {
    if (!node) return nullptr;
    unordered_map<Node*, Node*> visited;
    visited[node] = new Node(node->val);
    queue<Node*> q{{node}};
    while (!q.empty()) {
        Node* cur = q.front(); q.pop();
        for (auto& n : cur->neighbors) {
            if (!visited.count(n)) {
                visited[n] = new Node(n->val);
                q.push(n);
            }
            visited[cur]->neighbors.push_back(visited[n]);
        }
    }
    return visited[node];
}
```

### 📚 学到的知识点
- **深拷贝 vs 浅拷贝**：深拷贝所有节点都是 new 出来的，修改新图不影响原图
- **哈希表记录映射**：`原节点 → 克隆节点`，避免重复创建
- **先存再递归**：先 `visited[node]=clone` 再遍历邻居，防止环导致死循环
- **DFS vs BFS 都能做**：DFS 递归简洁，BFS 队列无栈溢出风险

### ⚠️ 易错点 & 反思
- ❌ 忘记 `visited[node]=clone` 放在遍历邻居之前 → 有环时会死循环
- ❌ `visited` 不传引用 → 每次递归拷贝一份 map，永远找不到已克隆的节点
- ❌ 直接 `return node` → 浅拷贝，新图仍指向原节点
- 💡 **图 DFS 和树 DFS 的区别**：树没有环，不需要 visited；图有环，必须记录已访问节点

- [138] Copy List with Random Pointer — 链表的深拷贝，思路类似
- [133] Clone Graph — 图的深拷贝，map + DFS/BFS
