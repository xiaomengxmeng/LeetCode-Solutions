## [Easy] Binary Search (704)
### 🎯 核心思路
标准二分查找：每次取中间值比较，缩小一半搜索范围，O(log n)

### 🔑 关键代码片段
```cpp
int left = 0, right = nums.size() - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] > target)      right = mid - 1;
    else if (nums[mid] < target) left = mid + 1;
    else                         return mid;
}
return -1;
```

### 📚 学到的知识点
1. `mid = left + (right - left) / 2` 防止溢出，安全写法
2. `while (left <= right)` 退出时 `left > right`，区间为空
3. 找到 target 返回 mid，找不到返回 -1

### ⚠️ 易错点 & 反思
- `else if(nums[mid] > target)` 和 `else if(nums[mid] > target)` 条件写重
- 循环结束后必须 `return -1`
- `right` 初始值是 `size() - 1` 不是 `size()`

## [Easy] Search Insert Position (35)
### 🎯 核心思路
704 模板 + 找不到时返回 left（插入位置）。循环结束时 left 是第一个 >= target 的位置。

### 🔑 关键代码片段
```cpp
int left = 0, right = nums.size() - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] > target)      right = mid - 1;
    else if (nums[mid] < target) left = mid + 1;
    else                         return mid;
}
return left;  // 插入位置
```

### 📚 学到的知识点
1. 704 模板不改变，只改最后返回值：`-1` → `left`
2. 循环结束时 left 是第一个 >= target 的位置，就是插入点
3. `return mid` vs `return left`：找到了当然要返回当前下标 mid

### ⚠️ 易错点 & 反思
- ❌ 找到了还 `return left` → left 是旧值，不是 mid
- ✅ 找到了 `return mid`，找不到才 `return left`

## [Medium] Find First and Last Position of Element in Sorted Array (34)
### 🎯 核心思路
用 lower_bound 找第一个 >= target 的位置（左边界），用 lower_bound(target+1)-1 找右边界。两次二分 O(log n)。

### 🔑 关键代码片段
```cpp
int lower_bound(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target)  right = mid - 1;
        else                      left = mid + 1;
    }
    return left;
}
vector<int> searchRange(vector<int>& nums, int target) {
    int L = lower_bound(nums, target);
    int R = lower_bound(nums, target + 1);
    if (L == R) return {-1, -1};
    return {L, R - 1};
}
```

### 📚 学到的知识点
1. **lower_bound 模板**：`>=` 时收缩 right，最终 left 是第一个 >= target 的位置
2. **右边界技巧**：找 target+1 的 lower_bound 再 -1，比单独写一个找右边界的函数更简洁
3. **配套判断**：`L == R` 说明 target 不存在，直接返回 [-1, -1]

### ⚠️ 易错点 & 反思
- ❌ lower_bound 中 if 条件写 `>` 而不是 `>=` → 找不到最左边的 target
- ❌ lower_bound 中等于时 return mid → 有重复值时不是最左位置
- ❌ 右边界忘记 -1 → 结果多 1
- ✅ `>=` 统一收缩 + 不 return mid，保证找到最左边界
