## [Easy] 爬楼梯 (70)

### 🎯 核心思路
斐波那契型 DP：状态定义 dp[i] 为爬到第 i 阶的方法数，转移方程 dp[i] = dp[i-1] + dp[i-2]（最后一步从 i-1 或 i-2 跨上来）

### 🔑 关键代码片段
```cpp
int climbStairs(int n) {
    if (n <= 2) return n;
    vector<int> dp(n + 1, 0);
    dp[1] = 1, dp[2] = 2;
    for (int i = 3; i <= n; i++)
        dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}
```

### 📚 学到的知识点
- **DP 三要素**：状态定义（dp[i]）、转移方程（dp[i-1]+dp[i-2]）、初始条件（dp[1]=1, dp[2]=2）
- **最后一步思维**：只关注"最后一步从哪来"，无需思考中间过程
- **空间优化**：可用滚动变量（a, b, c）将 O(n) → O(1)

### ⚠️ 易错点 & 反思
- dp 数组大小应为 n+1，索引从 1 开始
- n≤2 的情况要提前处理，否则 dp[2] 赋值可能越界
- 与斐波那契数列的区别：dp[2]=2 而非 1

### 🔗 关联题目
- 509. Fibonacci Number（同类型基础）
- 746. Min Cost Climbing Stairs（加 cost 变体）

## [Easy] 使用最小花费爬楼梯 (746)

### 🎯 核心思路
最小花费 DP：dp[i] 表示爬到第 i 阶的最小花费，从 i-1 或 i-2 跨上来取较小值。可在第 0/1 阶免费开始。

### 🔑 关键代码片段
```cpp
int minCostClimbingStairs(vector<int>& cost) {
    vector<int> dp(cost.size() + 1, 0);
    dp[0] = dp[1] = 0;
    for (int i = 2; i <= cost.size(); i++) {
        dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2]);
    }
    return dp[cost.size()];
}
```

### 📚 学到的知识点
- **DAM 型 DP**：从求和（+）到求最值（min/min）的转变
- **初始化差异**：起点免费，dp[0]=dp[1]=0
- **顶部位置**："超过数组末尾" → dp[n] 即答案
- **与 70 题的核心区别**：代价 + 求最值 vs 方法数求和

### ⚠️ 易错点 & 反思
- ❌ **循环从 i=0 开始** → i-1/i-2 负索引越界，需从 i=2 开始
- ❌ 答案误写 `min(dp[n], dp[n-1])` → dp[n] 已经是超过末尾的值
- 与 70 题的区别梳理：70（求和） vs 746（取 min）

### 🔗 关联题目
- 70. Climbing Stairs（同框架，求和方法数）
- 198. House Robber（不能相邻 + 取 max）

## [Medium] 打家劫舍 (198)

### 🎯 核心思路
"选或不选"型 DP：dp[i] 表示前 i+1 家（0~i）能偷到的最高金额。偷第 i 家 → dp[i-2] + nums[i]；不偷 → dp[i-1] → 取 max。

### 🔑 关键代码片段
```cpp
int rob(vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    if (n == 1) return nums[0];
    vector<int> dp(n, 0);
    dp[0] = nums[0];
    dp[1] = max(nums[0], nums[1]);
    for (int i = 2; i < n; i++) {
        dp[i] = max(dp[i-2] + nums[i], dp[i-1]);
    }
    return dp[n-1];
}
```

### 📚 学到的知识点
- **递推三连的演变路径**：70（+求和）→ 746（min 求最值）→ 198（max 求最值 + 可选跳过）
- **"选或不选"模型**：可跳过元素时，dp[i] = max(选, 不选)
- **三题框架统一**：都是线性 DP，状态依赖前两个位置

### ⚠️ 易错点 & 反思
- ❌ `if (n = 0)` → 赋值不是比较，必须用 `==`
- ❌ `n == 1` 时 `dp[1] = max(nums[0], nums[1])` 越界 — 要提前返回
- ❌ `return dp[n]` → dp 大小为 n，应返回 `dp[n-1]`
- 空间可优化为滚动变量 O(1)（存 prev2, prev1, curr 三个变量）

- 746. Min Cost Climbing Stairs（同框架，求最小花费）
- 213. House Robber II（环状版本进阶）
