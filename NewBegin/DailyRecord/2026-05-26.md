## [Medium] Search in Rotated Sorted Array (33)
### 🎯 核心思路
旋转数组二分：先判断 mid 在左段还是右段，再判断 target 在哪段来决定收缩方向。O(log n)

### 🔑 关键代码片段
```cpp
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] == target) return mid;
    // 判断 mid 在左段还是右段
    if (nums[0] <= nums[mid]) {
        // mid 在左段（大段）
        // target 在左段 且 在 mid 左边 → 往左搜
        if (nums[0] <= target && target < nums[mid])
            right = mid - 1;
        else
            left = mid + 1;     // 否则往右搜
    } else {
        // mid 在右段（小段）
        // target 在右段 且 在 mid 右边 → 往右搜
        if (nums[0] > target && nums[mid] < target)
            left = mid + 1;
        else
            right = mid - 1;    // 否则往左搜
    }
}
return -1;
```

### 📚 学到的知识点
1. **旋转数组分段**：`nums[0] <= nums[mid]` 判断 mid 在左段，否则在右段
2. **段内判断**：左段内 `nums[0] <= target < nums[mid]` 才往左搜；右段内 `nums[mid] < target < nums[0]` 才往右搜
3. **等号处理**：`nums[0] <= nums[mid]` 和 `nums[0] <= target` 的等号不能少，否则漏掉边界

### ⚠️ 易错点 & 反思
- ❌ `nums[0] < nums[mid]` 漏等号 → mid=0 时误入 else 分支
- ❌ `nums[0] < target` 漏等号 → target=nums[0] 时找不到
- ❌ 右段分支条件写反 → 复制左段条件没改
- ❌ 旋转数组不能用标准二分直接比大小
