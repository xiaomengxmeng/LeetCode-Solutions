## [Medium] Find Minimum in Rotated Sorted Array (153)
### 🎯 核心思路
和 nums[right] 比较：mid 在左段则往右搜，在右段则往左搜。while (left < right)，最终 left==right 就是最小值。

### 🔑 关键代码片段
```cpp
int left = 0, right = nums.size() - 1;
while (left < right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] > nums[right])
        left = mid + 1;   // mid 在左段，最小在右边
    else
        right = mid;      // mid 在右段，最小在 mid 或左边
}
return nums[left];
```

### 📚 学到的知识点
1. **与 33 的区别**：33 需要 if/else 双分支判断段和 target 方向；153 只需一个条件
2. **while (left < right) 的妙用**：退出时 left==right，就是答案，无需额外判断
3. **right = mid 不是 mid-1**：因为 mid 本身可能就是最小值，不能跳过

### ⚠️ 易错点 & 反思
- ❌ `right = nums.size()` → `nums[mid] > nums[right]` 越界
- ✅ `right = nums.size() - 1`
- ❌ 和 nums[0] 比较：无旋转时 [1,2,3] 全部 nums[mid] > nums[0]，一直往右缩到越界
- ✅ 和 nums[right] 比较：天然处理了无旋转的情况

## [Medium] Search a 2D Matrix (74)
### 🎯 核心思路
把 m×n 矩阵拉平成 n×m 长度的一维数组，标准二分。mid → (mid/n, mid%n) 坐标映射。

### 🔑 关键代码片段
```cpp
int m = matrix.size(), n = matrix[0].size();
int left = 0, right = m * n - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;
    int val = matrix[mid / n][mid % n];
    if (val == target) return true;
    else if (val > target) right = mid - 1;
    else left = mid + 1;
}
return false;
```

### 📚 学到的知识点
1. **二维坐标映射**：`mid / n` 做行，`mid % n` 做列
2. **拉平技巧**：只要是每行连续递增（上一行末尾 < 下一行开头），就能当一维数组处理

### ⚠️ 易错点 & 反思
- ❌ `mid / m` 而不是 `mid / n` → m 是行数，n 才是每行长度
- Z 字形搜索从右上角出发，也适用于无跨行连续的二维矩阵

## [Medium] Koko Eating Bananas (875)
### 🎯 核心思路
"二分答案"：二分速度 K（1 ~ max(piles)），用 canEat(K) 验证能否在 H 小时内吃完。找最小可行 K，while(left < right)。

### 🔑 关键代码片段
```cpp
int left = 1, right = *max_element(piles.begin(), piles.end());
while (left < right) {
    int mid = left + (right - left) / 2;
    if (canEat(piles, mid, h))
        right = mid;
    else
        left = mid + 1;
}
return left;

bool canEat(vector<int>& piles, int K, int h) {
    int count = 0;
    for (int p : piles)
        count += (p + K - 1) / K;  // 向上取整
    return count <= h;
}
```

### 📚 学到的知识点
1. **二分答案模式**：当答案在一个连续范围内，可以用二分猜答案 + 验证函数判断可行性
2. **向上取整公式**：`(p + K - 1) / K`，比 if(p%K) 分支写法更简洁
3. **K 的范围**：最小 1（不能为 0），最大 max(piles)（1 小时 1 堆足够）
4. **while(left < right)**：找最小可行值，和 153 一样的模板

### ⚠️ 易错点 & 反思
- ❌ K=0 → 吃不完，左边界必须是 1
- ✅ `(p + K - 1) / K` 一行搞定余数处理
- 可以用 `long long` 防止 count 溢出
